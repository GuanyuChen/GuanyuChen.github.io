Chrome扩展与应用开发

初窥

Chrome是Google公司基于WebKit开发的一款浏览器，内嵌Google自行研发的V8引擎用来提高解释JavaScript的速度。同时Chrome还允许开发者自定义各种各样的扩展来扩充其功能，并支持HTML5编写桌面程序。以上特点使得Chrome成为前端开发领域最主流的浏览器和平台。

扩展

Chrome扩展是一系列文件的集合，包括：

  • HTML文件
  • CSS文件
  • JavaScript文件
  • 图片等静态文件
  • manifest.json(最重要)

Chrome通过读取manifest.json文件来获取扩展相关信息，包括扩展名称、版本、更新地址、请求权限、扩展的UI界面入口等。

字段定义：

  • name : 扩展的名称
  • version : 扩展的版本(最多可以由三个圆点分成四段，每段只能是数字，数字不能大于65535，不能以0开头(可以是0，不能是0123)，版本号左侧为高位，新版本号需要比旧版本号高)
  • description : 扩展的描述
  • icons : 扩展相关图标文件的位置
  • browser_action : 指定扩展图标放在Chrome工具栏中
    • default_icon : 定义了相应图标文件的位置
    • default_title : 当用户鼠标悬停与扩展图标上时显示的文字
    • default_popup : 当用户单击扩展图标时所显示的页面的文件的位置

应用

Chrome将其平台上的程序分为扩展和应用，并且使用同样的文件结构

扩展与浏览器结合紧密，更强调扩展浏览器功能。而应用无法像扩展一样轻易获取用户在浏览器中浏览的内容并修改，它更像一个独立的，与Chrome关联不大的程序。此时可以把Chrome看成一个开发环境而不是浏览器。

Chrome应用还分为 Hosted App(托管应用) 和 Packaged App(打包应用) 。Hosted App更像一个高级的书签，结构比较简单。而Packaged App将文件打包在一个，一般可以离线时使用，因为文件都打包在本地。

步骤

由上文我们已知Chrome扩展的文件结构为 HTML文件 , CSS文件 , js文件 , 图片等静态资源文件 , manifest.json .

在具备了以上文件结构之后，我们可以将扩展载入Chrome中运行。

  1. 依次点击功能栏最右边 –> 更多工具 –> 扩展程序，打开扩展程序页面
  2. 勾选右上角“开发者模式”
  3. 点击“加载已解压的扩展程序”，选择扩展所在的文件夹

Manifest文件格式

manifest.json文件是整个扩展的入口，可以告诉Chrome这个扩展的相关信息，是扩展必不可少的部分。

manifest.json使用JSON格式保存数据

JSON

JSON是一种通用的数据交换格式。它包含两种结构：key:value对的形式，名称和值之间用冒号(:)连接，多个key:value对之间用逗号连接，最后在整个对象两侧加上”{“,”}” ; 还有一种是值的有序集合，值与值之间用逗号连接，最后在整个数组两侧加上”[“,”]”

无论是对象形式还是数组形式，它们的值均可以是字符串、数字、对象、数组、布尔和null中的一种，也就是说，JSON可以嵌套。

manifest关键属性

Google官方文档中对于扩展和应用给出两个不同的Manifest介绍页面，这是因为有些属性只能扩展使用，而有些属性只能由应用使用，如果两者同时出现，会使Chrome困惑。

扩展的manifest.json必须包含name,version,manifest_version,且manifest_version值只能为2；对于应用来说，还必须包含app属性。

可选的常用的属性有很多，我们应该保留一份模板，在编写新扩展时直接填入新的值。

PS : 对于非自定义属性，JavaScript可以直接像读取对象属性那样读取或更改。
