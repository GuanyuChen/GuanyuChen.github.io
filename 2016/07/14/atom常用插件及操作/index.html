atom学习路线

笔者作为sublime的资深用户，对于atom早期发布的几个版本并没有多大兴趣。早期的atom没有表现出来多么优秀的特性，而且经常会出现卡顿的情况。所以，一部分开发者对它还是持观望态度的。

不过，在2016年一年，atom的发展速度是无与伦比的。首先，它作为一个开源项目就比sublime有很大的优势。其次，atom已经发展起来了一个成形的社区做为支柱。并且，在笔者最近学习的React整个路线的开发过程中，atom的插件体系是比sublime完整的。所以，笔者决定将开发工作逐渐迁移到atom上来。

这篇博客用来记录atom插件和快捷键的学习过程

插件安装

atom插件安装正常情况下是非常方便的，在File->Setting->Install就可以搜索插件并安装。

但当网络不能翻墙时，安装就会出错。这个时候我们可以选择本地安装。本地安装步骤如下：

  1. cd ~/.atom/packages
  2. git clone 库的URL
  3. cd 库的路径
  4. npm install

常用插件

Emmet

Emmet主要目的就是加快前端开发速度，包括代码片段和简写展开。

PS: 简写语法用Tab调用

Emmet特性

  • 简写支持嵌套
  • 简写支持分组
  • 简写支持乘法
  • 简写支持自增、自减、起序、编号

Emmet语法

文档初始化
  • html:5 或 ! (用于HTML5文档类型)
  • html:xt (用于XHTML过渡文档类型)
  • html:4s (用于HTML4严格文档类型)
代码简写

  • # id | . class | [] 属性 | {} 内容

    PS:若是不带父元素，默认为是隐形生成(就近原则)

  • 后代 | + 兄弟 | ^ 上级

  • () 分组 | $ 自增 | $@- 自减 | $@数字 起序

常用快捷键

在这里不得不吐槽atom的快捷键实在太多了，随着插件的增多，经常会出现快捷键占用的情况。可以在keymap.cson文件里自定义常用的快捷键。一般默认user级快捷键优先级最高
