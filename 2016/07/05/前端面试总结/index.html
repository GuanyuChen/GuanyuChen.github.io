JavaScript基础

现在能想起来的需要记住的JavaScript基础特性主要有以下几种：

变量提升与作用域

众所周知，在推出ES6之后，JavaScript声明变量的方式一共有三种：let,const,var。其中let和const关键字是ES6的新特性，分别声明了局部变量和常量。此时我们应该知道，let和const声明的变量不存在变量提升，而由var声明的变量存在变量提升。

变量提升的意思是：JavaScript代码运行时，会把变量和函数的声明提到作用域的顶端。

PS：变量提升只是把声明提前，而不会把赋值也提升上来

ES6引入块级作用域的概念，let关键字声明的变量作用域限定在块级(大括号内部)，只有在所定义的块级作用域内部才能访问。

而在ES5中，JavaScript是用 自执行的匿名函数 来实现块级作用域的。

下面进行总结：

  • var声明的变量存在变量提升，没有块级作用域，作用域一般由函数来划分
  • let、const声明的变量不存在变量提升，作用域被限定到块级(大括号内)
  • 存在变量提升的变量可以先使用，再声明，不会报错，因为变量已经存在作用域的顶部(但如果在赋值之间使用会报undefined)
  • let不能重复声明，会报错
  • 使用函数表达式定义的函数会函数声明连带函数体提升到方法体最顶部

如果变量在函数体内声明，它的作用域是 函数作用域 。否则，它就是 全局作用域 。变量将会在执行进入作用域时被创建。块不会定义新的作用域，只有函数声明和程序才可以。变量在创建的时候会被初始化为undefined。如果变量声明语句带有赋值操作，则赋值操作只有在被执行的时候才会发生，而不是创建的时候。

闭包和作用域链

作用域链

当某个函数第一次被调用时会创建一个执行环境（EC）以及相应的作用域链，作用域链是由最里层向最外层延伸的。意思就是说：在函数内部可以调用外部的变量，而在外部不能调用函数内部的变量。作用域的终点是全局执行环境。

闭包

闭包是指有权访问另一个函数作用域中变量的函数。创建闭包的常见方式就是在一个函数内部创建另外一个函数。

关于闭包，还有一个常见问题

解决的方法也是再次利用闭包

this对象

this对象是在运行时基于函数的执行环境绑定的（匿名函数中具有全局性）

因为每个函数被调用的时候其活动对象都会自动获取2个特殊的变量：this和arguments。内部函数搜索这两个变量只会在其活动对象内搜索为止，永远不可能直接访问外部函数中的这两个变量。当然可以使用闭包，将外部的this值保存在一个内部可以访问的变量中即可。

全局上下文中的this指向window对象，匿名函数中的this也指向window

原型链

JavaScript中所有对象都有原型。当对象收到一个消息时(属性查找或方法调用)，JS先尝试在自身属性查找，如果查不到就会将消息传给对象的原型并以此类推。

PS：ES5中用原型链来实现继承 ES6新增class关键字实现继承

ES6也为原型新增了一个_proto_属性

A.isPrototypeOf(B) // A是否为B的原型 返回true或false

ES5使用Object.create()构造原型

var b = Object.create(a);   // 构造一个以a为原型的对象
var b = Object.create(a,{age:{value:13}});  //构造一个以a为原型的有自己独特的属性的对象

Object.getPrototypeOf(); // 获取对象的原型

构造函数是创建原型链最常用的方法

每个构造函数都有一个特殊属性Prototype 指向函数的原型

原型链的真正形成靠的是_proto_，每个对象的proto属性指向它的创建对象的prototype

A.prototype = new B();  // 最简单的继承

PS：为函数的prototype属性增加属性时，不要使用对象字面量的方式，这样会切断原型链。可以用A.prototype.x = 1;来赋值。

ajax相关

原生JS实现

GET与POST区别：

  • GET比POST更快
  • GET把请求信息缀到URL后面，相同的URL会有相同的返回(防止GET的等幂访问可以在URL后面缀上一个独一无二的id)
  • GET可传送简单数据，大小在1kb左右；数据追加到url后面，所以GET传送的数据按照url参数的方式附加在请求行中的资源路径后面
  • GET会被浏览器缓存，存在安全问题
  • POST默认每次提交都是不同的
  • POST把数据作为HTTP消息的实体内容发送给服务器
  • POST传递数据量大

总之，GET效率高，数据量小，安全性低，会被缓存，POST反之。

PS : GET请求传递的参数一定要先encodeURIComponent编码；从URL截取下来的信息要decodeURIComponent解码

当异步时，还需要有一个onreadystatechange函数，写在open的前面

readyState

0: 请求未初始化
1: 服务器连接已建立
2: 请求已接收
3: 请求处理中
4: 请求已完成，且响应已就绪

jQuery实现

$.get()

$.get(url,[data],[func])
  • url为请求地址，data为请求数据的列表，callback为请求成功后的回调函数
  • 该函数接受两个参数，第一个为服务器返回的数据，第二个参数为服务器的状态，是可选参数。
  • 服务器返回的数据是字符串形式。

$.post()

$.post(url,[data],[func],[type])

type是向服务器请求的数据的类型，可以是html，xml，json等；如果不设置，则与$.get()一样，返回字符串

$.ajax()

$.getScript()

动态载入脚本文件

$.getScript(url,func); //可以跨域！

$.getJson()

$.getJson(url,[data],[func]); //也可以跨域！

http协议相关

主要特点：

  • 应用层，面向对象的协议
  • 简单快速，常用请求有GET，POST，HEAD(HEAD请求页面的首部，返回响应的标题，常用与搜索引擎中)
  • 无连接，每次连接只处理一个请求，服务器与客户端交互完毕之后马上断开
  • 无状态，不记录以前的操作
  • 基于TCP连接

http请求

http请求由三部分组成：请求行，消息报头，请求正文

请求行以方法符号开头，空格分隔，后面跟着请求的URL和协议版本

http响应

http响应由三部分组成：状态行，消息报头，响应正文

状态行以http协议的版本开头，空格分隔，后面接响应状态代码，状态代码文本

状态码类别：

  • 1XX：请求已接受，马上处理
  • 2XX：请求被成功接受
  • 3XX：重定向
  • 4XX：客户端错误
  • 5XX：服务器错误

常用状态码：

  • 200 客户端请求成功
  • 400 客户端请求有语法错误，服务器不能理解
  • 403 服务器收到请求，但拒绝提供服务
  • 404 请求资源不存在(错误的URL)
  • 500 服务器错误
  • 503 服务器当前无法处理请求，一段时间可能恢复正常

cookie session

cookie机制在客户端保持状态，session机制在服务器端保持状态。在服务器端保持状态也需要在客户端保持一个标识，所以session机制可能需要借助cookie机制。

cookie内容主要包括：名、值、过期时间、路径、域。路径和域构成了cookie的作用范围。不设置过期时间，cookie的生命周期为浏览器会话期间，关闭浏览器窗口，cookie就消失，叫做会话cookie，保存在内存里。而存储在硬盘上的cookie可以在两个浏览器进程间访问。

服务器为客户端创建一个session。先检查客户端请求中是否有sessionid，如果有，则检索出来；如果没有，则创建一个session，并返回客户端。一般返回客户端的sessionid在客户端由cookie保存。

cookie与session区别：

  • cookie数据存放在客户端，session数据存放在服务器
  • cookie安全性不高
  • session在一段时间内会保持在服务器上，当访问过多，会占用服务器资源
  • 单个cookie保存的数据一般不会超过4K，且许多浏览器都会限制一个站点最多保存20个cookie

PS：把登录信息等重要信息放在session中；其他信息如果保存，可以放在cookie中
